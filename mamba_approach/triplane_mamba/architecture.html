■ Component Deep Dives

Tri-Plane Bidirectional Scanning Global SSM

Instead of scanning purely along individual axes, TriPlaneMamba scans structured 2D planes bidirectionally (6 SSM passes total):

  • hw-plane (axial): (B·D, H·W, C)
  • dw-plane (coronal): (B·H, D·W, C)
  • dh-plane (sagittal): (B·W, D·H, C)

Outputs are summed and fused via 1×1×1 Conv + InstanceNorm + LeakyReLU.

Multi-Scale Depth Conv Local Depths

Acts as a parallel branch to the Tri-Plane SSM to capture local depth correlations.

Employs four parallel 3D convolutions with varying depth-axis dilation rates: [1, 2, 4, 8].

Kernel shape is (3, 1, 1), ensuring filtering acts strictly across slices, perfectly complementing the in-plane scans.

CBAM3D Attention Skip Gates

Applied to encoder skip connections before decoder concatenation. Filters out noisy/irrelevant features.

Channel Attention: GlobalAvgPool + GlobalMaxPool → shared MLP → sigmoid gate per channel.

Spatial Attention: Channel-wise avg+max → Conv3d(2→1) → sigmoid gate per voxel.

Compound Loss V2 Multi-Objective

Loss components used during training for structural and perceptual fidelity:

L = 1.0·wMAE + 0.2·SSIM + 0.05·Grad + 0.1·FFL
+ 0.4·DS₂_loss + 0.2·DS₃_loss

Deep supervision terms apply the same composite loss to the intermediate decoder outputs.

No Checkerboard Decoder Upsample

Transposed convolutions frequently produce checkerboard artifacts. TriPlaneMamba prevents this by utilizing:

F.interpolate(mode='trilinear')
+ Conv3d(kernel=1)

Provides smooth, high-fidelity spatial upsampling directly.

Memory Optimizations VRAM

  • Gradient Checkpointing: Recomputes activations during the backward pass in TriPlaneEncoderBlocks.
  • Sequential Processing: hw, dw, dh planes are scanned sequentially using explicit del commands to minimalize activation peaks.
  • AMP (fp16) usage.
